<p class="ot-eyebrow">What is OpenTunnel?</p>

## Great agents, stuck on one machine. Until now.

<p class="ot-lede">Agents are brilliant on the machine they run on. The moment the task lives on another machine (a server, a build box, your homelab) they hit a wall of SSH keys, firewall rules, and standing credentials. Permanent infrastructure for a temporary need.</p>

<p class="ot-support">OpenTunnel is an ephemeral, end-to-end encrypted command tunnel built for exactly this moment. You start one foreground process on the remote machine and paste the printed prompt into your agent. From then on, the agent runs commands there like any other tool call: it sends a command, and stdout, stderr, and the real exit code come back as if the machine were local.</p>

<p class="ot-support">There is nothing to install and nothing to provision. No SSH keys to create and distribute, no inbound ports to open, no account to register. A relay routes the encrypted traffic between the two sides, but it cannot read it and it stores nothing. When the task is done, you press Ctrl+C: the session ends, the invite expires, and no trace of the access remains.</p>

</div>

<div class="ot-section-head">

<p class="ot-eyebrow">How it works</p>

## Three steps, zero infrastructure

</div>

<Steps>

1. **Start a session on the remote machine.**

   ```bash
   curl -fsSL https://opentunnel.sh | sh
   ```

   A temporary CLI is downloaded, checksum-verified, and opens one foreground session.

2. **Paste the generated prompt into your agent.**

   Your agent now has a remote shell as a tool. It runs commands and gets stdout, stderr, and exit codes back as if they were local tool calls.

3. **Press Ctrl+C when you're done.**

   The session ends, access is revoked, the temporary CLI is gone. Nothing persists, not on your machine and not on the relay.

</Steps>

</ContainerSection>

<p class="ot-eyebrow">Security</p>

## You don't have to trust the relay.

<p class="ot-lede">The relay routes opaque, encrypted frames between your agent and the remote machine. It cannot read your traffic, so it doesn't matter who operates it.</p>

<p class="ot-eyebrow">Deployment</p>

## Use mine, or run your own

<p class="ot-lede">Because the relay only ever sees ciphertext, the hosted relay is safe to use as-is. And because it needs no database, no accounts, and no persistent state, self-hosting is one command.</p>

<p class="ot-eyebrow">Philosophy</p>

## Deliberately small

<p class="ot-lede">OpenTunnel keeps the access model temporary and narrow on purpose. That is the security model, not a missing feature list. No accounts, no daemons, no audit logs (because there is nothing to log), no PTY, no file transfer, one client, one command at a time.</p>
