Strong Decisions Already In The Plan

1

Foreground Lifecycle

create owns the tunnel lifetime. Ctrl+C is revocation. No daemon, dashboard, stop/list/logs, or persistent session state.

2

Relay-Blind Runtime

The relay serves artifacts and routes packets, but never sees invite secrets, commands, output, exit codes, host metadata, or client identity as plaintext.

3

Go Monobinary

One opentunnel binary supports relay, create, and exec, with a required Noise spike before full implementation.

4

Repeated Sequential Exec

The invite is reusable while the foreground host session is alive, but only one client and one command may be active at a time.

Main Brainstorming Pressure Points

Security Model Sharpness

Noise pattern choice, invite semantics, relay-visible routing handles, replay boundaries, and what claims the product can safely make.

V1 Scope Control

The plan is coherent but large. It may need a thinner first implementation slice without violating the intended product experience.

/cli Distribution

Frictionless curl-to-temp-binary UX versus artifact version compatibility, temp cache safety, and honest supply-chain messaging.
